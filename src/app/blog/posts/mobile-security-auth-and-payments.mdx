## Security is a product feature

In fintech and healthcare apps, security is part of the core user promise. Weak authentication or fragile payment flows can erase trust immediately, no matter how good the UI looks.

## Principles I follow

- **Least privilege everywhere**: scope tokens and API access as tightly as possible.
- **Strong auth options**: combine secure session handling with biometric and social sign-in where relevant.
- **Sensitive data discipline**: avoid over-collecting data and encrypt what must be stored.
- **Failure-safe payments**: design idempotent payment flows with strong retry and reconciliation logic.

## Implementation details that reduce risk

- strict server-side validation for all sensitive actions
- signed requests and defensive API contracts
- robust audit logs for high-risk operations
- clear error messaging that helps users recover safely

## Outcome

By treating security as a continuous engineering responsibility, I delivered mobile products with reliable authentication and payment experiences while keeping friction low for end users.
